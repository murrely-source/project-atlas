# Solaris Skills Library

## SK-001 — Approved Master Workflow
Check the DHF, locate the latest approved master, confirm its version, and build only from that source.

## SK-002 — Brand Compliance
Use approved Solaris assets, colors, typography, terminology, and website. Do not crop the Brand Guide or create substitute logos.

## SK-003 — Dashboard Engineering
A dashboard must answer:
- What happened?
- What matters?
- What requires action?
- What is next?

## SK-004 — Intelligence Workflow
Research package → Chair review → Board discussion → Archive / Monitor / Integrate → Engineering.

## SK-005 — Information Validation
Classify content as fact, analysis, recommendation, opinion, or hypothesis. Do not mix categories without labels.

## SK-006 — Research and RAG Procedure
Use authoritative sources, compare evidence, identify agreement and disagreement, cite sources, and summarize neutrally.

## SK-007 — UI Review
Validate functionality, consistency, accessibility, responsiveness, brand compliance, and executive quality.

## SK-008 — DHF Review
Before major work, inspect related ADRs, Research Records, Approved Masters, current sprint, pending decisions, and risks.

## SK-009 — Executive Writing
Use a calm, clear, evidence-based, neutral, and purposeful tone.

## SK-010 — Human Firewall™
Assess awareness, appropriate reliance, verification, decision quality, trust, communication, escalation, readiness, and culture.

## SK-011 — Software Mindset
Define the user's job and workflow before deciding how the page should look.

## SK-012 — Quality Gate
Engineering Review • Brand Review • Governance Review • Executive Review.

## SK-013 — No Shortcuts
Do not substitute screenshots, approximations, or rushed placeholders for production assets and engineered solutions.

## SK-014 — Debugging and Validation
1. Reproduce the issue.
2. Record the expected and actual behavior.
3. Verify data and inputs.
4. Isolate the smallest failing component.
5. Identify the root cause.
6. Implement the smallest safe correction.
7. Retest the affected workflow.
8. Run regression checks.
9. Document the finding and corrective action.

Human review is mandatory before closing material defects.

## SK-015 — Source Accuracy
Verify dates, names, standards status, regulations, and claims with authoritative current sources before publication.

## SK-016 — UI Component Discipline
Build reusable headers, navigation, buttons, cards, tables, forms, status pills, filters, and chart containers.

## SK-018 — Summary-to-Detail Navigation
A topic receives a dedicated page when it needs multiple charts, filters, search, historical data, a distinct workflow, or substantial source material. Every overview chart must link to its detail workspace.
